How to Connect

Connect your Android phone to this dashboard in under 2 minutes. Both devices must be on the same WiFi network.

Phone Sender URL
http://{{ ip }}:8000/sender
Open this URL in Chrome on your Android phone. Your laptop IP was auto-detected. If you're on a different subnet, replace with your laptop's IP address.
cloud_sync
Render Cloud Sync
Session data auto-pushes to your Render deployment when a session ends
How it works: When a local session ends, Motion Studio automatically POSTs all readings to your Render app's API. You can then open your Render URL from any device to view full session history and replays.
Step-by-step setup
1

Same WiFi

Make sure your phone and this laptop are connected to the same WiFi network. Hotspot will also work if your phone is the hotspot and the laptop connects to it.

2

Open the sender page

On your Android phone, open Chrome and navigate to:

http://{{ ip }}:8000/sender
Alternatively scan a QR code or type the URL manually.
3

Enable DeviceOrientation in Chrome

Chrome blocks sensor access on plain HTTP. You must whitelist the server URL once:

chrome://flags/#unsafely-treat-insecure-origin-as-secure

Paste the flag URL into Chrome's address bar on your phone. In the text field that appears, enter:

http://{{ ip }}:8000
Change the dropdown to Enabled, then tap Relaunch. You only need to do this once per device.
4

Start streaming

Return to the sender page, tap Start. The button turns into a stop button and you'll see orientation values updating. The dashboard status badge will show Connected.

Troubleshooting

Stuck at CONNECTING

The phone can't reach the server. Check: same WiFi, correct IP, Windows Firewall allows port 8000, server is running.

Connected but values are 0

The Chrome flag wasn't set. Complete Step 3 exactly — the flag must be Enabled and Chrome relaunched before going to the sender page.

Wrong IP address shown

The server may have detected a VPN or secondary adapter IP. Find your real WiFi IP via ipconfig in PowerShell and use that instead.

Data streams but lags

WiFi congestion or a weak signal. Move closer to the router, or reduce send rate on the sender page (interval setting).
